## 氣泡排序法
基本原理
在排序的過程中，數值較大（或較小）的元素會透過不斷的交換，比較相鄰的兩個元素，並把大的往右邊推，所以一輪比較過後，當前陣列最大的元素一定會在最右邊

操作分析

第一回合（Pass 1）：從陣列的第一個元素開始，與第二個元素比較。如果第一個大於第二個，就交換它們。接著比較第二和第三個，依此類推，直到比較到陣列的最後一個元素。此時，全陣列最大的數字已經被安置在最後一個位置。

第二回合（Pass 2）：重複上述動作，從頭開始兩兩比較。但因為最後一個元素已經是最大值（已固定），所以這次只需要比較到倒數第二個位置。結束後，第二大的數字會被安置在倒數第二個位置。
重複執行：每一回合都會讓一個剩餘的最大值歸位，因此每一回合需要比較的次數都會比上一回合少一次。
結束條件：當經過一整輪的比較，完全沒有發生任何交換動作時，就代表所有元素都已經在正確的位置上，排序完成
## 插入排序法
基本原理

插入排序法的核心思想是將陣列分為兩個區塊：「已排序區（Sorted）」與「未排序區（Unsorted）」。
一開始，我們假設陣列的「第一個元素」本身就是一個已經排序好的序列。接著，我們逐一從「未排序區」拿出元素，將它與「已排序區」的元素從後往前進行比較，並插入到正確的位置。

操作分析

1.初始狀態：將陣列的第0個元素視為「已排序區」，第1到第n-1個元素視為「未排序區」。

2.取出目標（Target）：從未排序區取出第一個元素作為「目標值」，準備將它插入已排序區。

3.由後向前比較：將目標值與已排序區的元素（從最右邊最大的開始）逐一比較：如果已排序區的元素大於目標值，就把該元素往右移動一格，騰出空間。如果已排序區的元素小於或等於目標值，或者已經比對到陣列的最左邊（索引為 0），就停止比較。

4.執行插入：將目標值放入最後騰出來的那個空位中。此時，已排序區的長度增加1，未排序區減少1。

5.重複執行：重複步驟 2 到 4，直到未排序區沒有任何元素為止。
## 選擇排序法
基本原理

選擇排序法同樣會將陣列分為**「已排序區（Sorted）」與「未排序區（Unsorted）」。
它的核心動作是：在每一回合中，從「未排序區」裡面完整掃描**一遍，找出其中的最小值（或最大值），然後將這個最小值與「未排序區的第一個元素」進行交換，藉此將該元素納入「已排序區」。

操作分析

1.初始狀態：一開始，整個陣列都是「未排序區」，「已排序區」為空。

2.尋找最小值：從未排序區的第一個元素開始，向後掃描所有的元素，記錄下數值最小的元素所在的位置（Index）。

3.執行交換：掃描結束後，將找到的「最小值」與「未排序區的第一個元素」互換位置。經過這次交換，這個最小值就來到了陣列的最前面，成為「已排序區」的第一個元素。

4.縮小範圍：「已排序區」的長度增加1，而「未排序區」的起點往右移動一格（長度減少1）。

5.重複執行：重複步驟 2 到 4，每次都在縮小後的未排序區中尋找最小值並交換，直到未排序區只剩下一個元素（此時它必定是最大值），排序即告完成。
## 合併排序法
基本原理

「分而治之」的概念很簡單：當一個問題太龐大難以直接解決時，我們就把它切分成許多個小問題；如果小問題還是太難，就繼續切分，直到問題小到可以直接解決為止。解決完所有小問題後，再把它們的答案「合併」起來，大問題就解決了。
在排序的場景中：
一個元素的陣列，本身就已經是「排序好」的狀態。
將兩個「已經排序好」的陣列合併成一個大的排序陣列，是一件非常簡單且快速的事情。

操作分析

合併排序法的執行過程可以嚴格分為兩個階段：「分割（Divide）」與「合併（Merge）」。

1.分割階段（Divide）：
將原始陣列從正中間切成兩半，變成兩個子陣列。
對這兩個子陣列繼續進行「對半切割」的動作。
不斷重複切割，直到所有的子陣列都只剩下 1 個元素為止。此時，分割階段結束。

2.合併階段（Merge）：
從這些只有 1 個元素的子陣列開始，兩兩配對準備合併。
如何合併兩個已排序的子陣列（重點）：
準備一個額外的暫存空間。
設定兩個指標，分別指向這兩個子陣列的第一個（也就是最小的）元素。
比較這兩個指標所指的元素，將較小的那個拿出來，放到暫存空間裡，然後該子陣列的指標往後移動一格。
重複比較與取出的動作，直到兩個子陣列的所有元素都被依序放入暫存空間。
隨著兩兩合併的進行，子陣列的長度會從 1 變成 2，再變成 4、8... 不斷長大。
當最後兩個巨大的子陣列合併成一個包含所有元素的陣列時，排序就徹底完成了。
## 堆積排序法
基本原理

邏輯上是樹，物理上是陣列：我們將一維陣列的元素，按照由上而下、由左至右的順序，想像成一棵二元樹。例如陣列的第 0 個元素是樹根（Root），第 1、2 個元素是它的左右小孩，以此類推。這種特性讓我們可以直接在陣列中透過簡單的數學公式算出父子節點的位置，而不需要額外的指標空間。

最大堆積（Max-Heap）的定義：在這棵樹中，任何一個「父節點」的值，都必定大於或等於它的「子節點」。因此，整棵樹的「樹根（Root，也就是陣列的第一個元素）」一定會是這堆資料中的最大值。

操作分析

1.階段一：建立最大堆積（Build Max-Heap）目標是將原本雜亂無章的陣列，調整成符合「最大堆積」規則的結構。我們從樹的「最後一個非葉子節點（也就是最後一個有小孩的節點）」開始，由下往上、由右至左檢查。如果發現子節點的值大於父節點，就將它們交換（Swap）。交換後可能會破壞下方的堆積結構，因此需要繼續向下檢查並調整（這個動作稱為 Heapify 或是 Sift-down）。當這個動作執行到樹根時，整棵樹就成為一個最大堆積，此時陣列的第一個元素必定是全域最大值。

2.階段二：排序與調整（Extract Max & Heapify）將樹根（最大值）與陣列的最後一個元素互換。此時，最大值就來到了陣列的最後面，這正是它在遞增排序中應該在的正確位置。接著，我們將這個已經排好的最大值「踢出」堆積的範圍（也就是將堆積的有效長度減 1）。因為剛才把陣列尾端的元素換到了樹根，現在樹根可能不符合最大堆積的規則了。所以我們針對新的樹根執行一次 Heapify（向下調整），讓剩餘的元素再次形成最大堆積，找出剩餘元素中的最大值。重複「交換至尾端 $\rightarrow$ 縮小範圍 $\rightarrow$ 向下調整」的動作，直到堆積中只剩下一個元素為止，排序即告完成。
## 複雜度
1.氣泡排序法

時間複雜度：最佳：O(n)。當資料已經是完全排序的狀態，搭配提早結束的機制（Flag），只需走訪一次確認沒有發生任何交換即可結束。最壞 / 平均：O(n^2)。需要進行兩層迴圈的比較與頻繁的元素交換。

空間複雜度：O(1)。只需要常數級別的額外空間來暫存交換用的變數，屬於原地排序（In-place）。
2.插入排序法 

時間複雜度：最佳：O(n)。資料已排序時，每次比較發現不用往前移動就能直接進入下一迴圈。最壞 / 平均：O(n^2)。資料呈現反向排序時，每個元素都需要移動最多次。

空間複雜度：O(1)。同樣屬於原地排序（In-place）。
3.選擇排序法 

時間複雜度：最佳 / 平均 / 最壞：皆為 O(n^2)。不論資料的初始狀態為何，它都會不厭其煩地走訪剩餘的所有未排序元素來尋找極值，無法提早結束。

空間複雜度：O(1)。原地排序（In-place）。
4.合併排序法 

時間複雜度：最佳 / 平均 / 最壞：皆為 O(log n)。分割的步驟需要 log n 的深度，而每一層的合併步驟需要掃描 n 個元素，整體效能表現非常穩定。

空間複雜度：O(n)。在合併的過程中，需要額外配置與原陣列大小相同的記憶體空間來暫存合併的結果，屬於非原地排序（Out-of-place）。
5.堆積排序法

時間複雜度：最佳 / 平均 / 最壞：皆為 O(nlog n)。建立初始堆積需要 O(n) 的時間，之後每次取出元素並重新調整堆積結構需要 O(log n)，共需執行 n 次。

空間複雜度：O(1)。可以直接利用原本的陣列空間來模擬二元樹並進行元素交換（In-place），這是它在空間優化上的一大優勢。

## 結論
資料只有1000筆時，五種演算法的誤差都在毫秒以內，感受不到差別；但當資料來到100000筆時，合併排序法和堆積排序法不到40毫秒就能完成，但是氣泡排序法需要花15秒以上的時間

但當資料已經排序的差不多的時候，插入排序法的速度是最快的，原因是因為插入排序遇到已排序的元素就會立刻停止比較，當資料幾乎排好時，它實際上是以接近 $O(n)$ 的線性時間在運作，加上它本身不需要額外的記憶體開銷與遞迴呼叫，常數時間（Constant Time）極小，因此在局部排序或微調時是無敵的；此外，選擇排序法的時間也大幅增加，因為她會把所有的數據重新掃民一遍尋找極值再去排序，所以比起氣泡排序法和插入排序法，選擇花費的時間更多
